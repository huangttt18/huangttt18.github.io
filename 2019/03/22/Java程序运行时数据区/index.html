Java程序运行时数据区

发表于 2019-03-22
字数统计: 336 | 阅读时长 ≈ 1

Java程序在运行时的数据区域划分是怎么样的？

线程独占

  • 程序计数器
    • 程序运行的行号指示器
    • 不会发生内存泄漏
    • 对于正在运行的Java程序，如果当前执行的是Java方法，那么它的值为当前执行的行号，如果当前执行的时native方法，那么它的值为undefined
  • 虚拟机栈
    • 由多个栈帧组成，每个栈帧中都存储这局部变量表，操作栈，动态链接，返回地址等数据
      • 局部变量表，方法执行过程中的所有变量
      • 操作数栈，入栈、出栈、复制、交换、产生、消费等变量
    • SOE
    • OOM
  • 本地方法栈

线程共享

  • MetaSpace，Java8之前被称为永久代（PermGen）
  • PermGen使用的为JVM内存，MetaSpace使用本地内存
  • Java堆
    • 对象实例的分配区域
    • GC的主要区域
JVM调优参数
  • -Xss，每个线程虚拟机栈的大小
  • -Xms，堆的初始值
  • -Xmx，堆的最大值
内存分配策略
  • 静态存储，编译时确定每个数据目标在运行时的存储空间需求
  • 栈式存储，数据区需求在编译时未知，运行时模块入口前确定
  • 堆式存储，编译时或运行时模块入口都无法确定，动态分配
垃圾收集器
Redis持久化机制
